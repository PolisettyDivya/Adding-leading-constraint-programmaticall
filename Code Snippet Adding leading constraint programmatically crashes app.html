Adding leading constraint programmatically crashes app

nolngStackOverflow

Download
View thread

QUESTION

I'm trying to get my head around how adding constraints programmatically works. So far I have my code like so:

1import UIKit
2
3class ViewController: UIViewController {
4
5    override func viewDidLoad() {
6        super.viewDidLoad()
7        // Do any additional setup after loading the view, typically from a nib.
8        //addViewStandard()
9        addConstraintsView()
10    }
11
12    func addConstraintsView() {
13        let someView = UIView(frame: CGRect.zero)
14        someView.backgroundColor = UIColor.blue
15        // I want to mimic a frame set of CGRect(x: 20, y: 50, width: 50, height: 50)
16        let widthConstraint = NSLayoutConstraint(item: someView, attribute: .width, relatedBy: .equal, toItem: nil, attribute: .notAnAttribute, multiplier: 1.0, constant: 50)
17        let heightConstraint = NSLayoutConstraint(item: someView, attribute: .height, relatedBy: .equal, toItem: nil, attribute: .notAnAttribute, multiplier: 1.0, constant: 50)
18        let leadingConstraint = NSLayoutConstraint(item: someView, attribute: .leading, relatedBy: .equal, toItem: view, attribute: .leading, multiplier: 1.0, constant: 20)
19        someView.translatesAutoresizingMaskIntoConstraints = false
20        someView.addConstraints([widthConstraint, heightConstraint, leadingConstraint])
21        view.addSubview(someView)
22    }
23}
24

Now when I run the app it crashes because of the leading constraint. The error message is "Impossible to set up layout with view hierarchy unprepared for constraint". What am I doing wrong here? Should I be adding the constraints to the object (the blue box on this case) or adding them to its superview?

EDIT:

After code changes I have:

1func addConstraintsView() {
2        let someView = UIView(frame: CGRect.zero)
3        someView.backgroundColor = UIColor.blue
4        view.addSubview(someView)
5        someView.translatesAutoresizingMaskIntoConstraints = false

8        let leadingConstraint = NSLayoutConstraint(item: someView, attribute: .leading, relatedBy: .equal, toItem: view, attribute: .left, multiplier: 1.0, constant: 20)
9        someView.addConstraints([widthConstraint, heightConstraint])
10        view.addConstraints([leadingConstraint])
11    }
12

ANSWER

First of all,

1view.addSubview(someView)
2someView.translatesAutoresizingMaskIntoConstraints = false
3

should come before the constraints phase; you have to apply the constraints AFTER someView is added to its superview.

Also, if you are targeting iOS 9, I'd advise you to use layout anchors like 

1let widthConstraint = someView.widthAnchor.constraint(equalToConstant: 50.0)
2let heightConstraint = someView.heightAnchor.constraint(equalToConstant: 50.0)
3let leadingConstraint = someView.leadingAnchor.constraint(equalTo: view.leadingAnchor, constant: 20.0)
4NSLayoutConstraint.activate([widthConstraint, heightConstraint, leadingConstraint])
5

This way you don't have to worry about which view to apply the constraints to.

Finally (and to clear up your doubt), if you can't use layout anchors, you should add the leading constraint to the superview, not the view.
